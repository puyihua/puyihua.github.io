MacOS Terminal Proxy with Shadowsocks

0001-01-01 122 words 1 min read

Table of Contents

Because of some regulations, it becomes more and more difficult to visit with abroad sites in China. The throughput with abroad IP is limited within a very low level, which is nearly unusable.

Accordingly, I got some connect errors when I use some tools such as pip and conda to install packages.

To solve this problem, I started to think whether I can use Shadowsocks as proxy for my Terminal. And luckily, I find some informing tutorials from Google.

  • https://github.com/mrdulin/blog/issues/18
  • https://wido.me/sunteya/understand-bashrc-and-profile
  • https://stackoverflow.com/questions/38794015/pythons-requests-missing-dependencies-for-socks-support-when-using-socks5-fro

Concretely, add proxy=... and unproxy=... in ~/.bashrc. To ensure pip can use SOCKS5 proxy, type pip install requests[socks], and type proxy to start our high-speed terminal experience. Btw, using curl ip.cn to make sure your Terminal is on the proxy.
