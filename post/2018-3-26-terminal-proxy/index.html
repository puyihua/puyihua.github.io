MacOS Terminal Proxy with Shadowsocks

2019-01-02 187 words 1 min read

Table of Contents

Because of the Great Firewall, it becomes kind of impossible to access most of the abroad website without proxy in China. I always got a ‘404’ even when I was visiting certain abroad universities’ site. I used a advanced VPN tool shadowsocks to build a encrypted proxy between my laptop in China and my Linux Server in Japan. All of my http requests would go through Japan and then arrive America or Europe. In other word, I made a detour to avoid the firewall.

But there are still some problems when I was using Terminal instead of Browsers. I always got some connect errors when I use some tools such as pip and conda to download packages from abroad sources.

To solve this problem, I started to think whether I can use Shadowsocks as proxy for my Terminal. And luckily, I find some informing tutorials from Google.

  • https://github.com/mrdulin/blog/issues/18
  • https://wido.me/sunteya/understand-bashrc-and-profile
  • https://stackoverflow.com/questions/38794015/pythons-requests-missing-dependencies-for-socks-support-when-using-socks5-fro

Concretely, add proxy=... and unproxy=... in ~/.bashrc. To ensure pip can use SOCKS5 proxy, pip install requests[socks], and proxy to start the terminal proxy. Btw, using curl ip.cn to make sure your Terminal is on the proxy.
